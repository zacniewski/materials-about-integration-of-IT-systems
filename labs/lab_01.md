# Laboratorium 1: Git i GitHub w praktyce projektowej

## Czas trwania: 10 godzin

### Cel:
Opanowanie systemu kontroli wersji Git oraz platformy GitHub w stopniu pozwalającym na efektywną pracę indywidualną i zespołową.

### Zadania i ćwiczenia:
1. **Konfiguracja środowiska (1h):**
   - Instalacja Git.
   - Konfiguracja nazwy użytkownika i emaila.
   - Generowanie i dodawanie kluczy SSH do konta GitHub.

2. **Praca z lokalnym repozytorium (2h):**
   - Inicjalizacja projektu, śledzenie plików (`add`), tworzenie rewizji (`commit`).
   - Ignorowanie plików (`.gitignore`) – praktyczne przykłady dla różnych języków.
   - Analiza historii zmian (`log`, `diff`, `show`).

3. **Gałęzie i scalanie (3h):**
   - Tworzenie i przełączanie się między gałęziami (`branch`, `checkout`/`switch`).
   - Scalanie zmian (`merge`).
   - Praktyczne rozwiązywanie konfliktów.

4. **Współpraca z GitHub (4h):**
   - Tworzenie zdalnych repozytoriów.
   - Operacje `push`, `pull`, `fetch`.
   - Praca w zespole: Forkowanie, tworzenie Pull Requestów, przegląd kodu (Code Review).
   - Wykorzystanie GitHub Issues i projektów do zarządzania zadaniami.

### Wymagania na zaliczenie:
- Utworzenie publicznego repozytorium na GitHub.
- Wykazanie się poprawną historią commitów (znaczące opisy).
- Skuteczne wykonanie i rozliczenie Pull Requesta (może być w parach).
